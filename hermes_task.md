# Codex++ 开发任务

## 项目概述
中医智慧平台（中医治疗方案查询与分享平台）
项目路径：D:\Backup\Documents\中医易经

## 现有资产
- `产品开发文档.md` — 完整的产品需求文档（功能、数据库设计、API接口、前端页面结构等）
- `demo.html` / `demo.js` — 现有静态 Demo 前端
- `build_html.py` — 静态页面生成脚本
- `gen_data.py` — 测试数据生成脚本
- `_data.json` — 治疗方案数据

## 任务目标
根据 `产品开发文档.md` 构建一个可用的 Web 版中医智慧平台，在本地可运行。

## 技术要求
- **后端**：Python Flask/FastAPI + SQLite（方便本地运行，无需额外数据库服务）
- **前端**：基于现有 demo.html 样式风格改造，使用 Jinja2 模板或纯 HTML+JS 与后端 API 通信
- **认证**：JWT token 登录/注册
- **API**：按产品文档 API 接口清单实现核心接口

## 必须实现的核心功能
1. **用户注册/登录** — 手机号+密码，JWT 认证
2. **疾病分类浏览** — 13 科疾病分类，按分类查看疾病列表
3. **治疗方案查询** — 按疾病查看治疗方案列表（预览），按名称搜索
4. **治疗方案详情** — 预览+付费解锁完整内容（模拟支付，无需真实支付）
5. **疗效反馈** — 已购买用户可提交评分/疗效/描述，查看反馈聚合统计
6. **内容上传** — 用户可粘贴文字上传，模拟AI处理流程
7. **审核后台** — 管理员登录后可查看待审核列表、通过/驳回
8. **个人中心** — 积分查看、收藏、已购记录

## 数据
使用现有 `_data.json` 中的治疗方案数据作为初始数据，按文档的数据库设计迁移到 SQLite。

## 启动方式
项目完成后应提供一键启动脚本（run.py 或类似方式），单命令即可启动后端+前端。

## 额外要求
- 代码清晰，有必要的注释
- 中文界面
- 兼容现有 demo.html 的视觉风格（暖色调、中医风格）
